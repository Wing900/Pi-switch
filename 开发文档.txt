# Pi Switch 开发文档

## 1. 产品定位

Pi Switch 是一个轻量级桌面工具，用于配置 piAgent 的不同模型提供商，并自动获取可用模型，最后写入 piAgent 配置并启动 piAgent。

它只做一件事：

> 配置 Provider → 获取模型 → 选择默认模型 → 写入 Pi 配置 → 启动 Pi

Pi Switch 不是新的 Agent，不管理 MCP，不做聊天，不做 Prompt，不做知识库。

---

## 2. 核心用户场景

用户经常遇到的问题：

1. 不同 Provider 的 baseUrl、apiKey、model 参数很难记。
2. Pi 自定义 provider 需要手动写 `models.json`，麻烦。
3. 不知道 API Key 是否配置成功。
4. 不知道当前 Pi 默认使用哪个 provider 和 model。
5. 想快速切换 DeepSeek、OpenRouter、Ollama、LM Studio、自建 OpenAI 兼容接口。

Pi Switch 解决这些问题：

* 用图形界面填写 provider 参数
* 自动请求 `/models` 获取模型列表
* 用户勾选模型后自动生成 Pi 配置
* 一键设置默认模型
* 一键启动 piAgent

---

## 3. 产品边界

### 3.1 做什么

* 添加 Provider
* 编辑 Provider
* 删除 Provider
* 测试连接
* 自动获取模型列表
* 选择默认模型
* 写入 piAgent 的配置文件
* 启动 piAgent
* 检查 API Key 环境变量是否存在

### 3.2 不做什么

* 不保存 API Key 明文
* 不做 MCP 管理
* 不做聊天窗口
* 不做模型价格比较
* 不做复杂 benchmark
* 不做账号登录系统
* 不做云同步
* 不做 Electron 大体积应用

---

## 4. 技术栈

推荐技术栈：

* Go
* Wails v2
* Vanilla TypeScript
* 原生 CSS
* JSON 文件存储

不使用：

* Electron
* React
* Vue
* Ant Design
* Tailwind
* 后端 Web Server

原因：

* 软件功能简单，不需要大型前端框架
* Wails 使用系统 WebView，比 Electron 轻
* Go 负责配置、HTTP、文件读写、启动子进程
* 前端只做表单和按钮

```text

+------------------------------------------+
| Set the following model as Pi default?   |
|                                          |
| Provider: DeepSeek                       |
| Model: deepseek-chat                     |
|                                          |
| This will update:                        |
| ~/.pi/agent/settings.json                |
|                                          |
|                  [Cancel] [Confirm]      |
+------------------------------------------+
```

确认后写入：

```json
{
  "defaultProvider": "deepseek",
  "defaultModel": "deepseek-chat",
  "defaultThinkingLevel": "off"
}
```

注意：写入时不能覆盖用户其他字段，只能合并更新。

## 16. 配置文件设计

### 16.1 Pi Switch 自己的配置

路径：

```text
~/.piswitch/config.json
```

格式：

```json
{
  "version": 1,
  "providers": [
    {
      "id": "deepseek",
      "name": "DeepSeek",
      "type": "openai-compatible",
      "baseUrl": "https://api.deepseek.com/v1",
      "apiKeyEnv": "DEEPSEEK_API_KEY",
      "apiKeyLiteral": "",
      "api": "openai-completions",
      "proxy": "",
      "headers": {},
      "models": [
        {
          "id": "deepseek-chat",
          "name": "deepseek-chat",
          "reasoning": false,
          "contextWindow": 128000,
          "maxTokens": 8192
        }
      ]
    }
  ],
  "settings": {
    "piCommand": "pi",
    "piSettingsPath": "",
    "piModelsPath": ""
  }
}
```

### 16.2 Pi models.json 输出

路径：

```text
~/.pi/agent/models.json
```

生成格式：

```json
{
  "providers": {
    "deepseek": {
      "baseUrl": "https://api.deepseek.com/v1",
      "api": "openai-completions",
      "apiKey": "$DEEPSEEK_API_KEY",
      "models": [
        {
          "id": "deepseek-chat",
          "name": "deepseek-chat",
          "reasoning": false,
          "contextWindow": 128000,
          "maxTokens": 8192
        }
      ]
    }
  }
}
```

如果是 Ollama：

```json
{
  "providers": {
    "ollama": {
      "baseUrl": "http://localhost:11434/v1",
      "api": "openai-completions",
      "apiKey": "ollama",
      "models": [
        {
          "id": "qwen2.5-coder:7b",
          "name": "qwen2.5-coder:7b",
          "reasoning": false
        }
      ]
    }
  }
}
```

### 16.3 Pi settings.json 输出

路径：

```text
~/.pi/agent/settings.json
```

只更新这些字段：

```json
{
  "defaultProvider": "deepseek",
  "defaultModel": "deepseek-chat",
  "defaultThinkingLevel": "off"
}
```

必须保留 settings.json 里已有的其他字段。

## 17. 自动获取模型设计

首版只支持 OpenAI-compatible `/models` 接口。

请求：

```http
GET {baseUrl}/models
Authorization: Bearer <apiKey>
```

如果使用 API Key 环境变量：

```text
apiKey = os.Getenv(apiKeyEnv)
```

如果使用本地服务，例如 Ollama / LM Studio：

```text
apiKey 可以为空
或使用 apiKeyLiteral
```

解析常见格式：

```json
{
  "data": [
    {
      "id": "deepseek-chat"
    },
    {
      "id": "deepseek-reasoner"
    }
  ]
}
```

转换为：

```json
[
  {
    "id": "deepseek-chat",
    "name": "deepseek-chat",
    "reasoning": false
  },
  {
    "id": "deepseek-reasoner",
    "name": "deepseek-reasoner",
    "reasoning": true
  }
]
```

reasoning 判断可以先用简单规则：

```text
model id 包含 reasoner / reasoning / r1 / thinking
→ reasoning = true
```

其他能力参数让用户手动编辑。


### 22.4 JSON 写入失败

显示：

```text

Please check file permissions.
```

## 23. 安全要求

1. 不保存 API Key 明文。
2. 不打印 API Key。
3. 日志中不能出现 Authorization header。
4. 写入 Pi 配置前自动备份。
5. 删除 Provider 前二次确认。
6. 只读写以下路径：

   * `~/.piswitch/config.json`
   * `~/.pi/agent/settings.json`
   * `~/.pi/agent/models.json`
7. 不扫描用户其他目录。
8. 不上传任何配置到网络。


## 29. 最终判断

Pi Switch 的首版不要做成“大而全的 Agent 管理器”。

最优形态是：

```text
一个轻量 Wails 桌面 GUI
+
一个 Go 后端配置生成器
+
一个 /models 自动获取器
+
一个 Pi 启动器
```

只要这个流程顺滑，它就已经有价值。
